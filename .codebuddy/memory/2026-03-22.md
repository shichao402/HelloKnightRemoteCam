# 2026-03-22

## 项目改造方案确定

HelloKnightRemoteCam 从"纯拍摄工具"升级为"作业管理 + 拍摄一体化平台"。

### 架构决策
- **四个进程**：Core Service（数据权威，24h常驻家庭服务器）、WeCom Agent（TS，企业微信长连接via OpenClaw）、Flutter客户端（PC按需启动）、手机服务端（不变）
- **通信方式**：Core↔Agent 用 MCP，Core↔Flutter 用 REST API + 事件推送（精确对接不需要MCP），Flutter↔手机 WebSocket（已有）
- **仓库结构**：monorepo，新增 `core/` 和 `agent/` 顶级目录
- **Core Service 技术栈**：用户倾向强类型语言，具体待定
- **开发顺序**：Phase 1 Core Service → Phase 2 Flutter任务模块 → Phase 3 MCP Server → Phase 4 WeCom Agent

### 关键设计决定
- PC端不是数据权威（不24h在线），Core Service才是
- Core Service 和 Agent 是独立进程，不合并
- Core Service 暴露两套接口：REST API（给Flutter）+ MCP Server（给AI Agent）
- 否决Web端，家长批阅通过WeCom Bot + 任务ID完成
- 企业微信用 OpenClaw 长连接模式（2026年3月企业微信新支持的特性）
